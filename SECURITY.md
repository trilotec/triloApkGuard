# Security Policy

## Scope

This repository publishes a community edition of the TriloApkGuard core.

It is not a guarantee of complete hardening, and public code in this repository should not be treated as a full disclosure of any future commercial protection design.

## Reporting

If you discover a security issue in the public codebase, report it privately to the maintainer before public disclosure.

Until a dedicated security contact channel exists, use a private maintainer contact path rather than opening a public issue with exploit details.

## What To Include

Please include:

- affected file or component
- impact summary
- reproduction steps
- Android version / ROM details if relevant
- whether the issue affects build-time behavior, runtime behavior, or both

## Out Of Scope

The following are generally out of scope for this public repository policy:

- requests for stronger anti-reversing guarantees
- disclosure demands for private or future commercial key-management designs
- reports based solely on the claim that reverse engineering is still possible

## Response Expectations

Valid reports should be triaged for:

- correctness
- exploitability
- compatibility impact
- whether the issue belongs in the public core or only in a private extension
